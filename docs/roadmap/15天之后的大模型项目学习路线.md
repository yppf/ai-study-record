# 15天之后的大模型项目学习路线

## 1. 先说结论

如果你已经按前面的 `15天大模型数学基础到主线衔接学习计划` 学完了，那么：

- `数学和结构基础基本够了`
- `不建议继续深挖更多数学细节`
- `下一步应该转向大模型主线、工程能力和项目`

更准确地说：

- 对“理解大模型核心原理”来说，这套数学基础已经够扎实
- 对“转行去做大模型岗位”来说，还差 `工程 + 项目 + 面试表达`

所以后面的重点不再是：

- 再补更多抽象数学

而是：

- 把数学基础接到真实大模型工作流

---

## 2. 你现在已经具备了什么

学完前面的 15 天后，你理论上应该已经具备这些能力：

- 能看懂张量、shape、广播、矩阵乘法
- 能理解梯度、反向传播、自动微分
- 能手推线性回归、softmax、交叉熵、两层 MLP 前向
- 能理解 attention、self-attention、Transformer block
- 能理解 tokenizer、embedding、next-token prediction
- 能理解 LoRA 和 RAG 的基本逻辑

这意味着你已经跨过了最容易卡人的阶段：

- “术语我听过，但其实不懂”

接下来你最该做的是：

- 把这些概念变成 `会跑代码、会做项目、会讲清楚`

---

## 3. 现在还缺什么

如果目标是大模型应用工程、算法工程或相关转行岗位，后面至少还缺这 5 块：

### 1. Hugging Face 和 Transformers 工程基础

你要会：

- 加载 tokenizer
- 加载模型
- 跑推理
- 看懂 `generate`
- 看懂模型输入输出

### 2. 微调主线

你要会：

- SFT 的基本流程
- LoRA / QLoRA 的基本使用
- 知道训练数据长什么样
- 知道 loss 曲线怎么观察

### 3. RAG 主线

你要会：

- 文档切分
- 向量化
- 检索
- rerank
- 拼 prompt
- 基础问答链路

### 4. Agent / 工作流主线

你要会：

- 工具调用
- 多步流程
- 状态管理
- 简单工作流编排

### 5. 项目和作品集

你至少要有：

- 一个 RAG 项目
- 一个微调或 Agent 项目

---

## 4. 后面的学习重点怎么取舍

如果你当前目标是转行，不建议继续把主要精力放在这些事情上：

- 继续系统学更多高等数学
- 花很多时间证明复杂公式
- 过早深挖分布式训练底层细节

更建议把时间花在这些地方：

1. `Transformers + Hugging Face`
2. `LoRA / QLoRA`
3. `RAG`
4. `Agent`
5. `做项目`

一句话就是：

`现在最缺的不是更多数学，而是把已有数学基础接到真实工程里。`

---

## 5. 推荐的后续 6 周路线

## 第 1 周：Transformers 和 Hugging Face 基础

### 目标

- 建立标准工程接口意识
- 能把 tokenizer、model、generate 串起来

### 必学内容

- `AutoTokenizer`
- `AutoModelForCausalLM`
- `generate`
- chat template
- model card

### 推荐资料

- Hugging Face Course  
  https://huggingface.co/learn/nlp-course

- Transformers 文档  
  https://huggingface.co/docs/transformers/index

- GitHub  
  https://github.com/huggingface/transformers

### 这一周的最低产出

- 跑通一个最小推理脚本
- 记录一份“输入 prompt -> token 数 -> 输出结果”的实验笔记

---

## 第 2 周：Tokenizer、生成、模型输入输出

### 目标

- 真正看懂大模型推理输入输出
- 弄清 tokenizer 和生成策略

### 必学内容

- token ids
- attention mask
- padding
- temperature
- top-k / top-p

### 推荐资料

- Hugging Face Tokenizers  
  https://github.com/huggingface/tokenizers

- Hugging Face Course  
  https://huggingface.co/learn/nlp-course

### 这一周的最低产出

- 自己写一个最小“文本 -> token -> generate -> 输出”的脚本
- 自己实验 temperature 和 top-k 的差异

---

## 第 3 周：LoRA / QLoRA 微调入门

### 目标

- 会跑最小微调流程
- 理解参数高效微调的实际价值

### 必学内容

- LoRA
- QLoRA
- 训练数据格式
- 训练参数
- loss 观察

### 推荐资料

- PEFT 文档  
  https://huggingface.co/docs/peft/index

- GitHub  
  https://github.com/huggingface/peft

- self-llm  
  https://github.com/datawhalechina/self-llm

- LLaMA-Factory  
  https://github.com/hiyouga/LLaMA-Factory

### 这一周的最低产出

- 跑通一个 LoRA 微调 demo
- 写下训练参数、显存占用、loss 变化

---

## 第 4 周：RAG 入门项目

### 目标

- 能独立搭一个最小 RAG
- 理解为什么很多岗位会高频问 RAG

### 必学内容

- chunk
- embedding
- retrieval
- rerank
- prompt augmentation

### 推荐资料

- RAG from Scratch  
  https://github.com/langchain-ai/rag-from-scratch

- FlagEmbedding  
  https://github.com/FlagOpen/FlagEmbedding

- LlamaIndex  
  https://github.com/run-llama/llama_index

### 这一周的最低产出

- 做一个本地文档问答 demo
- 最好有一个 README 写清楚流程

---

## 第 5 周：Agent 或工作流应用

### 目标

- 把“单轮问答”扩展成“多步执行”
- 形成第二个作品方向

### 可选方向

- Agent 工具调用
- 多步工作流
- 文档分析助手
- 简单数据问答助手

### 推荐资料

- Hugging Face Agents Course  
  https://huggingface.co/learn/agents-course/zh-CN/unit0/introduction

- GitHub  
  https://github.com/huggingface/agents-course

- LangGraph  
  https://github.com/langchain-ai/langgraph

- smolagents  
  https://github.com/huggingface/smolagents

### 这一周的最低产出

- 做一个最小 Agent 或工作流 demo
- 能清楚说明“输入 -> 工具 -> 输出”的链路

---

## 第 6 周：项目整理、简历表达、面试准备

### 目标

- 把零散学习成果变成可投递材料

### 必做内容

- 补 README
- 补技术选型说明
- 补运行步骤
- 补效果截图
- 补踩坑总结

### 你至少应该整理出两个项目

1. `RAG 项目`
2. `LoRA 微调项目` 或 `Agent 项目`

### 这一周的最低产出

- 2 个可展示项目
- 1 份能讲清楚的项目介绍
- 1 版转行向简历

## 8. 最少要做的两个项目

## 项目 1：RAG 项目

### 推荐题目

- 本地知识库问答
- PDF 文档助手
- 课程文档问答助手

### 最低要求

- 文档导入
- 分块
- 向量检索
- 检索增强回答
- README

### 面试时要能讲清楚

- 为什么要分块
- 为什么选这个 embedding 模型
- 为什么需要 rerank
- 你怎么评估回答质量

---

## 项目 2：LoRA 或 Agent 项目

### 可选方向 A：LoRA 微调

- 某个垂直任务微调
- 指令数据微调
- 领域问答风格微调

### 可选方向 B：Agent / 工作流

- 工具调用助手
- 搜索 + 总结助手
- 文档分析助手

### 面试时要能讲清楚

- 为什么选 LoRA 而不是全参数微调
- 为什么 Agent 需要工作流而不是单轮 prompt
- 你做过哪些调试和优化

---

## 9. 学到什么程度就可以开始投递

如果你达到了下面这些标准，就已经比较接近可以投递了：

- 能讲清楚 Transformer 和 next-token prediction
- 能跑通 Hugging Face 基础推理
- 能跑通一个 LoRA demo
- 能做出一个最小 RAG 项目
- 至少有 2 个项目可以展示
- 能把“数学 -> 模型 -> 工程 -> 项目”串起来表达

这时候你即使不是“资深大模型工程师”，也已经不是“只会调 API”的状态了。

---

## 10. 最后一句建议

学完前面的数学基础之后，最常见的误区是：

- 继续收藏资料
- 继续看概念
- 一直不做项目

更好的路线是：

`数学地基 -> 主线工程 -> 两个项目 -> 简历与面试`

如果你后面能一直沿着这条线走，转行成功率会比“只看教程不落项目”高很多。
