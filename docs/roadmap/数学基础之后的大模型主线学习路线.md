# 数学基础之后的大模型主线学习路线

## 1. 这份路线解决什么问题

前一份《大模型数学基础学习总结》解决的是地基问题：

- 你能看懂张量、梯度、softmax、交叉熵、attention
- 你能把数学公式和模型训练代码对应起来

这份路线解决的是主线问题：

- 数学学完以后，下一步到底该怎么进大模型
- 哪些内容是岗位主线，哪些只是补充
- 每个阶段该看什么 GitHub 仓库、什么学习资料

---

## 2. 先说结论

如果你想转行到大模型方向，数学之后最应该补的是这 8 块：

1. Transformer 与 LLM 基础
2. Tokenizer、Embedding、数据与预训练目标
3. Hugging Face 基础工程能力
4. SFT 与 LoRA / QLoRA
5. DPO / RLHF / GRPO 等后训练
6. 推理部署与服务化
7. RAG
8. Agent

建议按 `8 周主线` 来学，而不是一上来到处收集资料。

---

## 3. 8 周主线学习顺序

### 第 1-2 周：Transformer、Tokenizer、LLM 基础

这一阶段的目标：

- 把 Transformer 真正学明白
- 搞清楚 token、词表、BPE、SentencePiece、Embedding
- 能说清楚 GPT、BERT、T5 这种模型的大方向区别

你要重点掌握：

- self-attention、multi-head attention、position encoding
- causal mask
- tokenizer 为什么会影响模型能力和成本
- base model 和 instruct model 的区别

建议产出：

- 自己画一张 Transformer 结构图
- 自己写一页术语对照表：token、vocab、merge、embedding、logits、causal mask
- 能用自己的话解释 GPT 的 next-token prediction

推荐资料路径：

- Hugging Face LLM Course
  - 学习路径：https://huggingface.co/learn/nlp-course
  - GitHub：https://github.com/huggingface/course
- Hugging Face Tokenizers
  - GitHub：https://github.com/huggingface/tokenizers
- Datawhale Happy-LLM
  - GitHub：https://github.com/datawhalechina/happy-llm
- LLMBook 中文站
  - 在线阅读：https://llmbook-zh.github.io/
  - GitHub：https://github.com/datawhalechina/llmbook
- 从零实现 LLM
  - GitHub：https://github.com/rasbt/LLMs-from-scratch

推荐学法：

- 理论主线用 `Hugging Face LLM Course + Happy-LLM`
- 代码直觉用 `LLMs-from-scratch`
- tokenizer 单独配 `huggingface/tokenizers`

---

### 第 3 周：模型家族、预训练、后训练全景

这一阶段的目标：

- 把大模型主线图建立起来
- 不再只知道 GPT 一个名字
- 知道预训练、SFT、偏好优化在整个生命周期中的位置

你要重点掌握：

- Encoder-only、Decoder-only、Encoder-Decoder 的区别
- BERT、GPT、T5、LLaMA、Qwen 这些家族的大方向
- 预训练、指令微调、偏好优化、推理部署的整体流程

建议产出：

- 写一页“大模型生命周期流程图”
- 写一页“Base / Instruct / Chat / Reasoning”区别

推荐资料路径：

- Happy-LLM
  - GitHub：https://github.com/datawhalechina/happy-llm
- LLMBook
  - 在线阅读：https://llmbook-zh.github.io/
  - GitHub：https://github.com/datawhalechina/llmbook
- minimind 中文注释
  - GitHub：https://github.com/MLNLP-World/minimind-notes
- Hugging Face Transformers 文档
  - 文档：https://huggingface.co/docs/transformers/index
  - GitHub：https://github.com/huggingface/transformers

推荐学法：

- 用中文资料把全景串起来
- 用 `transformers` 文档把常见术语和工程接口接起来

---

### 第 4 周：Hugging Face 工程基础

这一阶段的目标：

- 具备最基本的大模型工程阅读能力
- 能看懂 `tokenizer -> model -> generate`
- 能用 Hugging Face 完成最小推理与实验

你要重点掌握：

- `AutoTokenizer`
- `AutoModelForCausalLM`
- `generate`
- model card、tokenizer config、chat template
- pipelines 与原生调用的区别

建议产出：

- 跑通一个本地或云端推理 demo
- 写一个最小推理脚本
- 记录输入 prompt、token 数、输出差异

推荐资料路径：

- Transformers 文档
  - 文档：https://huggingface.co/docs/transformers/index
  - GitHub：https://github.com/huggingface/transformers
- Hugging Face Course
  - 学习路径：https://huggingface.co/learn/nlp-course
  - GitHub：https://github.com/huggingface/course
- LitGPT
  - GitHub：https://github.com/Lightning-AI/litgpt

推荐学法：

- `Transformers` 用来建立“工程标准接口”意识
- `LitGPT` 用来看更清晰、偏训练工程的实现

---

### 第 5 周：SFT、LoRA、QLoRA

这一阶段的目标：

- 真正进入“模型能不能调”的阶段
- 理解参数高效微调为什么重要
- 能跑通一个基础微调流程

你要重点掌握：

- SFT 是什么
- LoRA / QLoRA 的基本原理
- adapter、rank、target modules 这些常见概念
- 全参数微调和参数高效微调的区别

建议产出：

- 跑通一个 LoRA 或 QLoRA 的 demo
- 记录训练参数、显存占用、loss 变化
- 自己总结“什么时候该用 LoRA”

推荐资料路径：

- PEFT 文档
  - 文档：https://huggingface.co/docs/peft/index
  - GitHub：https://github.com/huggingface/peft
- Datawhale self-llm
  - GitHub：https://github.com/datawhalechina/self-llm
- LLaMA-Factory
  - GitHub：https://github.com/hiyouga/LLaMA-Factory
- ms-swift
  - GitHub：https://github.com/modelscope/ms-swift
- LitGPT
  - GitHub：https://github.com/Lightning-AI/litgpt

使用建议：

- 初学微调，`self-llm` 很适合入门
- 想快速跑通多模型微调，优先看 `LLaMA-Factory` 或 `ms-swift`
- 想理解实现细节，继续看 `LitGPT`

注意：

- `LLaMA-Factory` 过去曾公开披露过旧版本安全问题，安装时不要照搬过时教程，直接使用当前版本与官方文档

---

### 第 6 周：DPO、RLHF、GRPO、评测

这一阶段的目标：

- 理解为什么 SFT 之后还要做偏好优化
- 知道 DPO、RLHF、GRPO 分别大致在解决什么问题
- 建立“训练之后必须评测”的意识

你要重点掌握：

- preference data 是什么
- DPO 的核心思想
- RLHF 的大致流程
- reasoning 类后训练为什么常提到 GRPO
- 基础评测框架怎么用

建议产出：

- 写一页 “SFT vs DPO vs RLHF vs GRPO” 对照表
- 跑一次现成评测脚本

推荐资料路径：

- TRL 文档
  - 文档：https://huggingface.co/docs/trl/index
  - GitHub：https://github.com/huggingface/trl
- Hugging Face Transformers 文档
  - 文档：https://huggingface.co/docs/transformers/index
- lm-evaluation-harness
  - GitHub：https://github.com/EleutherAI/lm-evaluation-harness
- OpenCompass
  - GitHub：https://github.com/open-compass/opencompass

推荐学法：

- `TRL` 负责建立后训练方法概念
- `lm-evaluation-harness` 建立英文主流评测视角
- `OpenCompass` 更适合中文环境和国产模型评测生态

---

### 第 7 周：推理部署与服务化

这一阶段的目标：

- 知道模型“会跑”和“能上线”不是一回事
- 了解推理吞吐、延迟、显存、量化、batching 的基本概念
- 能把模型变成一个服务

你要重点掌握：

- 本地推理与服务化推理的区别
- vLLM 的定位
- TGI 的定位
- llama.cpp 在 CPU / 轻量部署中的价值

建议产出：

- 跑通 1 个 OpenAI 兼容接口服务
- 对比 1 次不同推理后端的体验差异

推荐资料路径：

- vLLM
  - 文档：https://docs.vllm.ai/en/latest/
  - GitHub：https://github.com/vllm-project/vllm
- Text Generation Inference
  - GitHub：https://github.com/huggingface/text-generation-inference
- llama.cpp
  - GitHub：https://github.com/ggml-org/llama.cpp

推荐学法：

- 想走服务化部署，先看 `vLLM`
- 想看 Hugging Face 官方推理服务路线，看 `TGI`
- 想做本地化、轻量化、CPU 部署，看 `llama.cpp`

---

你要重点掌握：

- embedding、reranker、chunk、retriever
- 向量检索与重排序
- RAG 的基本链路
- Agent 的工具调用、状态管理、工作流编排

## 5. 按岗位方向怎么取舍

### 如果你目标是大模型应用工程师

优先级建议：

1. Hugging Face 基础
2. SFT / LoRA / QLoRA
3. RAG
4. Agent
5. 部署
6. 评测

这类岗位最重要的是：

- 跑通模型
- 做出应用
- 会调参
- 会排查常见问题

### 如果你目标是大模型训练 / 算法工程师

优先级建议：

1. Transformer 与预训练
2. Tokenizer 与数据
3. SFT / DPO / RLHF / GRPO
4. 评测
5. 分布式训练与优化
6. 部署

这类岗位更看重：

- 对模型结构和训练目标的理解
- 对数据、loss、训练策略的理解
- 对实验和评测的严谨性

### 如果你目标是大模型平台 / 推理部署工程师

优先级建议：

1. Transformers 基础
2. vLLM / TGI / llama.cpp
3. 量化与服务化
4. 评测与压测
5. 多模型管理

这类岗位更看重：

- 吞吐
- 延迟
- 显存利用率
- 部署稳定性

## 6. 学完这份主线后，能到什么水平

如果你按这条路线完整走完，并且每个阶段都做最小 demo，一般可以达到下面的水平：

- 能看懂大模型主流工程仓库的基本结构
- 能解释 Transformer、Tokenizer、SFT、LoRA、DPO、RAG、Agent 的基本逻辑
- 能用 Hugging Face 跑模型、做基础微调、做简单应用
- 能搭一个最小 RAG 或 Agent 项目
- 能把“数学基础”真正接到“岗位主线”

## 7. 这时候够转行了吗

### 结论

`对转应用型岗位，已经比较接近可投递状态。`

但还有一个关键差距：

`你需要项目。`

最少建议补 2 个项目：

1. 一个 RAG 项目
2. 一个微调或 Agent 项目

最好每个项目都具备：

- README
- 技术选型说明
- 运行方式
- 效果展示
- 你踩过的坑和调优记录

---

## 8. 建议你接下来的执行顺序

如果你刚学完数学基础，最建议这样接：

1. `第 1-2 周`：LLM Course + Happy-LLM + Tokenizers
2. `第 3-4 周`：Transformers + LitGPT + minimind-notes
3. `第 5-6 周`：PEFT + self-llm + TRL
4. `第 7 周`：vLLM / TGI / llama.cpp 三选一先跑通
5. `第 8 周`：RAG from Scratch + FlagEmbedding + Agents Course
6. `最后 2-3 周`：做项目并整理作品集

## 10. 最后一句建议

从转行视角看，最好的路线不是：

`疯狂看资料。`

而是：

`数学地基 -> 大模型主线 -> 2 个项目 -> 简历与面试准备`

只要你把这条线走顺，后面你在面试里就不只是“会用 API”，而是能真正说明白：

- 模型为什么这样设计
- 为什么这样训练
- 为什么这样部署
- 为什么这样做应用
