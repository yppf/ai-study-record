# 大模型方向数学基础学习总结

## 1. 目标

这份总结面向准备转行大模型方向的学习者，目标不是把数学学成纯理论体系，而是先做到下面 4 件事：

- 把和机器学习、大模型相关的数学知识串起来
- 建立基本的数学直觉
- 能手推常见核心公式
- 能把公式和代码、模型结构对应起来

建议采用 `中文主线 + 英文术语对照 + 代码验证` 的学习方式。

## 4. `d2l-zh` 重点目录怎么学

### `chapter_preliminaries`

这是数学和代码地基，必须认真学。

- `ndarray`：张量、shape、广播、切片
- `pandas`：表格数据预处理、缺失值、one-hot
- `linear-algebra`：向量、矩阵、点积、矩阵乘法、范数
- `calculus`：导数、偏导、梯度、链式法则
- `autograd`：自动微分、计算图、`backward`
- `probability`：条件概率、贝叶斯、期望、方差
- `lookup-api`：学会查文档

这一章的目标：

- 看懂张量和 shape
- 手推简单梯度
- 理解 `loss.backward()` 本质是什么

### `chapter_linear-networks`

这是你第一次把数学和训练流程连起来。

- `linear-regression`：线性回归、平方损失、梯度下降
- `linear-regression-scratch`：从零实现训练循环
- `linear-regression-concise`：理解框架封装
- `softmax-regression`：logits、softmax、交叉熵
- `image-classification-dataset`：分类数据读取与 shape
- `softmax-regression-scratch`：从零实现分类训练
- `softmax-regression-concise`：理解 `CrossEntropyLoss`

这一章的目标：

- 能手推线性回归和 softmax 回归
- 能理解 `输入 -> 线性层 -> 损失 -> 梯度 -> 更新`
- 能把 `softmax + cross-entropy` 和 LLM 输出概率联系起来

### `chapter_multilayer-perceptrons`

这章回答“为什么必须有非线性”“为什么会过拟合”“为什么深层网络能训练”。

- `mlp`：多层感知机与激活函数
- `mlp-scratch` / `mlp-concise`：从零实现与简洁实现
- `underfit-overfit`：欠拟合与过拟合
- `weight-decay`：权重衰减
- `dropout`：随机失活
- `backprop`：反向传播与链式法则
- `numerical-stability-and-init`：梯度消失、爆炸、初始化

这一章的目标：

- 理解非线性为什么重要
- 理解过拟合和正则化
- 能顺着链式法则看懂反向传播

### `chapter_attention-mechanisms`

这是从通用深度学习走向 Transformer 的桥。

- `attention-cues`：Query、Key、Value
- `nadaraya-waston`：把注意力理解成带权平均
- `attention-scoring-functions`：mask、加性注意力、缩放点积注意力
- `multihead-attention`：多头注意力
- `self-attention-and-positional-encoding`：自注意力和位置编码
- `transformer`：Transformer 编码器、解码器、FFN、残差、LayerNorm

这一章的目标：

- 看懂注意力本质是带权汇聚
- 看懂多头、自注意力、位置编码、因果 mask
- 看懂 Transformer block 的基本结构

## 6. 每天怎么学最有效

建议每天保持同一个节奏：

- `20 分钟`：读正文
- `20 分钟`：手推 1 个公式或 1 个 shape
- `30-50 分钟`：跑代码并改 1 个参数
- `10 分钟`：写笔记

建议长期维护 3 份笔记：

- `术语表`：中英对照
- `公式卡`：每张卡只记 1 个公式
- `错题本`：记录卡住的问题

## 8. 这些内容了解大模型够了吗

### 结论

`够打基础，但还不够胜任完整的大模型工作。`

更准确地说：

- 如果你的目标是 `看懂大模型核心原理`，这些内容已经是非常扎实的基础
- 如果你的目标是 `转行做大模型工程/算法岗位`，这些内容还不够，需要继续补充

### 这些内容已经解决了什么

- 你不再害怕公式
- 你能看懂 loss、gradient、softmax、attention 这些高频概念
- 你能理解 Transformer 为什么成立
- 你能开始读 BERT、GPT、LLaMA 一类模型的结构文章

### 还缺什么

如果你要真正进入大模型方向，后面至少还要补这 6 块：

#### 1. Transformer 和 GPT/BERT 系列模型

- Encoder-only、Decoder-only、Encoder-Decoder 的区别
- BERT、GPT、T5、LLaMA 的基本结构差异
- 预训练、微调、推理的流程

#### 2. NLP 与 Tokenization 基础

- Token、词表、BPE、SentencePiece
- Embedding 的作用
- 语言建模目标

#### 3. 大模型训练与微调

- SFT
- LoRA / QLoRA
- DPO / RLHF 的基本概念
- Prompt tuning、Prefix tuning 的基本认识

#### 4. 大模型应用工程

- RAG
- 向量数据库
- Agent
- 工具调用
- 工作流编排

#### 5. PyTorch 和工程能力

- 读懂训练脚本
- 看日志、看 loss 曲线
- Debug shape、显存、梯度问题
- 使用 Hugging Face 生态

#### 6. 评测与部署

- 离线评测和在线评测
- 幻觉、鲁棒性、安全性
- 推理部署、量化、吞吐、延迟

## 10. 最终建议

如果你现在是准备转行的大模型从业者，这套内容非常值得学，而且应该认真学，因为它会决定你后面看模型、看训练、看论文时是不是“只会背术语”。

但也要避免一个误区：

`不要以为学完这些数学，就等于学完了大模型。`

更准确的定位应该是：

- 这套内容是你的 `数学地基`
- Transformer、GPT、微调、RAG、Agent 才是你的 `岗位主线`

最理想的节奏是：

- 用 `6 周` 打数学和模型基础
- 再用 `6-8 周` 专门补大模型原理与工程实践

这样转行会更稳，也更容易在面试里说清楚“我为什么理解这个模型，而不只是会调 API”。
